The following is summary you made in another chat for a project we're continuing here, with some edits from me.

We are building a wizard themed board game called Arcane. Many of the themes, enemies, factions, and items come from Dresden Files, but there is influence from various other sources.

The project has four phases:
1 Implement the full game logic in code.
2 Run automated simulations to evaluate card balance.
3 Generate printable card assets.
4 Build a tabletop simulator mod version of the game.

We're currently in phase 1.

Within phase 1 there are 9 steps:
1. Core engine and console app
2. Prep and battle phases
3. Spells
4. Enemies
5. Schools and resistances
6. Artifact 
7. Tools 
8. Potions 
9. Factions

We are currently on step 5.

1. Core engine and console app
The project currently runs as a c# console application, but the game logic is contained inside a Core engine so the same engine could later support a graphical UI or Simulator where all user actions are automated and all game output is hidden

The architecture roughly follows this structure:

UI (Console) -> Commands -> Game Engine -> Events -> Console Renderer

2. Prep and battle phases
The game alternates between Prep Phase and Battle Phase.

Prep Phase
Prep phase has 3 rounds. Each prep round allows 5 player actions. Mana is restored at the start of each prep round. Knowledge does not reset between rounds. It accumulates and is used to purchase spells from the market.

Typical prep actions include:
Train – spend mana to increase Max Mana
Rest – heal HP
Learn – gain knowledge used to buy spells
Buy Spell – purchase spells from the market

Battle Phase
After prep rounds end, a battle begins.

During battle the player can take actions such as:
Cast Spell
Channel (gain mana)

The player gets 3 actions, and then all monsters attack

3. Spells
Each spell contains properties such as:
Name
Spell School
Knowledge Cost
Mana Cost
Target Type
Damage
Heal
Status Effects
Special flags (such as OncePerBattle)

Damage and healing values are handled through a Value system.

The Value system supports three types:
Flat values
Dice rolls (example 2d6)
Percent values (example 50%)

Values can be created from strings such as:
"3"
"2d6"
"25%"

Dice rolls generate both a result and a list of rolls so the console output can show the full roll breakdown.

Spell effects supported so far include:
Single target damage
Area damage
Cleave damage
Healing
Mana generation
Lifesteal
Burn (damage over time)
Freeze (skip next attack)

Status effects are stored on monsters and tick at the end of the monster attack phase.

Market System
There is a spell market where players can purchase spells.

The market works as follows:
Six spells are available at a time.
At the start of each prep round the entire shop refreshes.
Purchased spells are removed from the market.
Spells are drawn from the SpellLibrary.

4. Enemies
Each monster currently has:
Name
Health
Damage
Theme
Threat value (internal only)
Status effects

Monster themes categorize monsters into types:
Beasts
Goblins
Ogres
Undead
Elemental
Warlocks
Demons
Vampires

Example monsters include:
Goblin
Goblin Shamanka
Goblin King
Werewolf
Spider
Wyvern
Unicorn
Chimera
Dragon
etc.

Encounters
Battles spawn groups of monsters based on a theme.
Monster selection currently uses threat values to construct encounters appropriate for the current round.

5. Schools and resistances
Spell schools have been implemented but monster resistances/weaknesses/immunities have not.
The schools are currently:
Arcane
Fire
Ice
Lightning
Earth
Holy
Blood

6. Artifact
Artifacts are passive items that give buffs, either earned from battle or awarded from faction reputation

7. Tools
Tools are similar to artifacts but they are first learned in the market with knowledge, and then require mana upkeep during prep to be active for the battle phase.

8. Potions
Recipes are bought in the market, brewed in prep phase, and used in battle

9. Factions
Gain (or lose) reputation via actions in both prep and battle phases. Unlock rewards, could be spells, artifacts, other buffs or bonuses.
Factions for now are
White Council (Arcane) led by The Merlin
Summer Court of Faerie (Fire) led by Queen Titania
Winter Court of Faerie (Ice) led by Queen Mab
The Church (Holy), including the Knights of the Cross
The Circle, Denarians, Vampire Courts, or The Outsiders (Blood) - undecided
Lightning and Earth currently do not have a faction

Playtesting notes:
- The game currently encourages repeating the same few optimal spells because monsters do not yet react differently to different spell schools.
- Enemy damage is currently high relative to player health. We need to implement shield spells and artifacts/tools before working on balance though.
- With AoE Freeze spells, mana generation spells, and 3 actions between each enemy attack, all encouters are solved. Monster freeze immunity solves some of this but we also need to use a combination of giving freeze spells much higher mana cost, OncePerBattle flag, or keep them single target.
